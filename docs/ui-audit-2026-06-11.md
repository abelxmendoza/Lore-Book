## The core finding

The app's calm core already exists — the top of the sidebar (Chat, Characters, Locations,
Memories, All Entries, Groups) is exactly the right product. The overwhelm is everything below
the fold: Skills, Love, Gossip & claims, Timeline, Saga, Discovery, Quests, Lore book, Lore
editor, Photos, Entity resolution. A new user scrolling the nav meets eleven more destinations,
most of which render empty states or internal tooling.

## Bugs found during audit (fix regardless of triage)
1. **Characters book renders empty with data present** — repro: load `/characters` fresh; shows
   "0 total" while DB has characters. Suspect the list fetch fails silently or filters exclude
   all rows (check default filter vs `relationship_depth`).
2. **Self-inflicted 429** — Memories surface fires enough parallel requests on mount to trip our
   own `aiRateLimit`/rateLimit middleware; users see "Too many requests" on first visit.
3. **Onboarding modal** blocks every surface until dismissed (by design, but it fires on every
   fresh device/profile and fully covers mobile).

## Recommended sequence
1. **Quick wins, no product decisions:** route-level `React.lazy` (main chunk should drop well
   under 1 MB; charts/pdf libs load only where used), fix the two bugs above, soften chat
   empty-state copy.
2. **The calm shell (needs cut-list sign-off):** sidebar → keep-list + collapsed "Labs" section
   driven by a settings toggle; settings consolidation; merge entry-browsing surfaces.
3. **Mobile polish on keepers:** Memories filter chips → a single "Filters" sheet; big detail
   modals → full-screen pages on mobile.
4. **Phase 3 homepage** after the shell is calm (per canonical blueprint).
